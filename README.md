# AI Fake Or Real

> Crowdsourced AI content detection for any URL on the internet вҖ” plus a simple public API.

A Chrome extension **and** developer API that combine **community votes**, an
**AI-comment heuristic**, and **AI analysis** to tell you whether content was
generated by AI.

## Features

- **One-glance badge** вҖ” green (Human), yellow (Mixed), or red (AI) on every tab
- **Crowdsourced voting** вҖ” vote Human / Mixed / AI Generated on any page
- **AI text analysis** вҖ” statistical heuristics detect AI writing patterns (perplexity, burstiness, vocabulary richness, sentence uniformity)
- **Comment accusation heuristic** вҖ” detects when other users call content AI ("this is AI generated", "AI slop"), precisely (ignores benign mentions and negations)
- **Multi-platform** вҖ” Reddit (native JSON), Instagram/Facebook postsВ·reelsВ·storiesВ·videosВ·photos, YouTube, TikTok, X, and most sites via Open Graph
- **Combined scoring** вҖ” AI + crowd, dynamically weighted by vote count
- **Public API** вҖ” one call (`GET /api/v1/check?url=`) returns a verdict any app can use; free API keys with usage metering

## Public API (for other apps)

```bash
curl "http://localhost:8000/api/v1/check?url=https://www.reddit.com/r/x/comments/a/t/"
# -> { "verdict": "ai_generated", "ai_probability": 0.82, "confidence": "high", ... }
```

One call returns a `verdict` (`human` / `mixed` / `ai_generated` / `unknown`)
plus the underlying signals. Create a free API key with `POST /api/v1/keys`.
Full guide: **[docs/API.md](docs/API.md)** В· interactive docs at `/docs`.

## Quick Start

### Prerequisites

- Docker & Docker Compose
- Node.js 20+

### Backend

```bash
docker-compose up
```

The API will be available at `http://localhost:8000`. Interactive docs at `http://localhost:8000/docs`.

### Extension (Chrome, Edge, Brave, and Firefox)

```bash
cd extension
npm install
npm run build:all      # builds dist/ (Chromium) and dist-firefox/ (Firefox)
```

**Chrome / Edge / Brave** (any Chromium browser):
1. Go to `chrome://extensions`
2. Enable "Developer mode" вҶ’ "Load unpacked"
3. Select the `extension/dist` folder

**Firefox:**
1. Go to `about:debugging#/runtime/this-firefox`
2. "Load Temporary Add-onвҖҰ" вҶ’ select `extension/dist-firefox/manifest.json`

The extension uses the cross-browser WebExtension APIs (`browser`/`chrome`), so
the same source builds for both. The API server URL is configurable on the
options page and shared across the popup and background worker.

#### How analysis works (reads the page you're viewing)
When you click **Analyze**, the content script reads the page you're already
looking at вҖ” expanding collapsed comments on Reddit, Instagram, and Facebook вҖ”
and sends the caption + comments to the backend. This works behind login walls
and on SPAs that a server can't fetch. If the content script can't run (e.g. a
restricted page), it falls back to server-side fetching. The
[comment-accusation heuristic](docs/API.md) then flags content other users have
called out as AI.

> Toolbar icons are generated (no design assets needed). Regenerate them with
> `python3 extension/scripts/generate_icons.py`.

## Testing

```bash
cd backend
pip install -r requirements-dev.txt
python -m pytest          # runs against in-memory SQLite

cd ../extension
npm run typecheck         # tsc --noEmit
```

Tests cover the scoring math, AI text-detection heuristics, the cache's
graceful degradation, and every API route (success, validation, auth, and
error cases). No PostgreSQL or Redis instance is required to run them.

## Reliability & Security

- **Rate limiting** вҖ” per-IP fixed-window limits on `vote` and `analyze`
  (Redis-backed, in-memory fallback, fails open on backend errors).
- **SSRF protection** вҖ” `/analyze` refuses to fetch private, loopback,
  link-local, or cloud-metadata addresses, and only follows http(s).
- **Input validation** вҖ” URLs must be http(s) and within a length cap.
- **Bounded fetches** вҖ” page downloads are capped (default 2 MB).
- **Caching** вҖ” score reads are cached in Redis with graceful degradation.
- **Observability** вҖ” every response carries an `X-Request-ID`; access logs
  include method, path, status, and latency.
- **`/health`** вҖ” verifies database connectivity for load balancers/probes.

All limits and toggles are configurable via environment variables
(see `backend/.env.example`).
