# Monetization Ideas

Current plan: **Free.** Build a useful app, get users, grow.

We'll collect monetization ideas here for when the time comes.

## Ideas

| Idea | Notes |
|------|-------|
| BYOK (bring your own keys) | **Shipped.** Users add their own provider keys in Settings; their checks run on their own quota, off the shared keys. Funds nothing directly but removes cost for power users. |
| Usage-based API pricing | Free tier generous; charge high-volume API consumers. Metering already built. |
| Pro tier (extension) | More detailed analysis, priority scanning, vision/video, export reports. Funds paid provider quota. Surface only in options page. |
| Enterprise API | Bulk analysis, webhooks, SLA. Separate pricing page. |

See [scaling.md](scaling.md) for the cost model and where the free-tier ceilings are.

## Principles

- Never gate the core verdict behind a paywall
- Never show monetization in the popup or in-page overlays
- Free tier must be genuinely useful, not a teaser
- Growth first, monetization later
